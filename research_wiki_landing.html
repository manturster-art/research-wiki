The problem문제 인식

LLMs cite papers that don't exist.LLM은 존재하지 않는 논문을 인용합니다.

Ask any LLM about your research area and it will confidently cite authors, years, and journals — most of which are fabricated.

연구 분야를 LLM에 질문하면 저자, 연도, 저널명을 자신 있게 제시합니다. 그 중 상당수는 만들어낸 것입니다.

🌀

Hallucinated citations환각 인용 (Hallucinated citation)

GPT-4 and similar models invent plausible-sounding references. In academic writing this is catastrophic — you can't cite a paper that doesn't exist.GPT-4 같은 모델은 그럴듯한 참고문헌을 지어냅니다. 학술 글쓰기에서 이는 치명적입니다 — 존재하지 않는 논문을 인용할 수는 없으니까요.

🔍

Untraceable claims추적 불가능한 주장

Even when citations are real, it's hard to know which exact passage supports which claim. Every answer should be traceable to a specific paper and page.인용이 실제라도 어떤 구절이 어떤 주장을 뒷받침하는지 알기 어렵습니다. 모든 답변은 특정 논문의 특정 페이지까지 추적 가능해야 합니다.

📂

Context overload컨텍스트 한계

As your literature grows, feeding 50+ PDFs into a single prompt becomes impractical. You need a structured, searchable store that grows with your research.문헌이 늘어날수록 PDF 50편을 한 번에 넣는 것은 비현실적입니다. 연구와 함께 성장하는 구조화된 지식 저장소가 필요합니다.

✅

research-wiki solves thisresearch-wiki가 해결합니다

Every wiki page is extracted from a real PDF you own. Every answer cites only papers in your wiki. Every entry is cross-verified against two independent metadata databases.모든 위키 페이지는 실제 보유한 PDF에서 추출됩니다. 모든 답변은 위키에 있는 논문만 인용합니다. 모든 항목은 두 개의 독립 메타데이터 DB로 교차 검증됩니다.

Architecture구조

Three layers. One ground truth.3개 계층, 하나의 원본.

Every piece of knowledge flows from an immutable PDF through structured summaries to a cross-linked wiki page.

모든 지식은 불변 PDF에서 구조화된 요약을 거쳐 상호 링크된 위키 페이지로 흐릅니다.

Verification검증

Four sources. Zero guessing.4개 출처. 추측 제로.

Every entry is cross-checked against two independent academic databases and the original PDF — automatically.

모든 항목은 두 개의 독립 학술 DB와 원본 PDF를 대조해 자동으로 검증됩니다.

Wiki YAML

Frontmatter extracted directly from the PDF: title, authors, year, DOI, category, dissertation mapping.PDF에서 직접 추출한 frontmatter: 제목, 저자, 연도, DOI, 카테고리, 논문 매핑.

OpenAlex

250M+ works database. Confirms DOI, author list, cited-by count, Open Access PDF URL. 30-day local cache.2억 5천만 건 이상의 학술 DB. DOI, 저자 목록, 피인용수, OA PDF URL 확인. 30일 로컬 캐시.

Crossref

Independent second database. When OpenAlex and Crossref agree, trust is high. When they disagree, the PDF decides.OpenAlex와 독립적인 두 번째 정본. 두 DB가 일치하면 신뢰도 상승. 불일치 시 PDF가 최종 판정.

Installation설치 방법

Up and running in 5 minutes.5분이면 시작할 수 있습니다.

No build tools. No package manager. Pure Node.js and Python — both already on your machine.

빌드 도구 불필요. 패키지 매니저 불필요. Node.js와 Python만 있으면 됩니다.

Commands명령어 목록

enrich_wiki.js

Full verification run전수 검증 실행

Verifies all wiki entries against OpenAlex. Detects metadata drift, discovers new DOIs, reports downloadable OA PDFs.전체 위키 항목을 OpenAlex에 대조합니다. 메타데이터 drift 감지, 누락 DOI 발견, 다운로드 가능한 OA PDF 보고.

node _workspace/enrich_wiki.js
crossref_helper.js

Independent cross-check독립 교차 검증

Query Crossref as a second ground truth. crosscheck mode calls both OpenAlex and Crossref simultaneously and compares results.Crossref를 두 번째 정본으로 조회합니다. crosscheck 모드는 두 DB를 동시에 호출해 결과를 비교합니다.

Interactive dashboard인터랙티브 대시보드

Graph health check그래프 구조 분석

wiki-verifier agent

4-way / 5-way verification agent4-way / 5-way 검증 에이전트

A project-local Claude Code agent. 4-way for English (wiki YAML ↔ OpenAlex ↔ Crossref ↔ PDF) and 5-way for Korean (+ KCI Open API). Uses dual-write: verification_log.md (immutable history) + review_queue.md (actionable items) — never auto-edits wiki files.로컬 Claude Code 에이전트. 영어 자료는 4-way (wiki YAML ↔ OpenAlex ↔ Crossref ↔ PDF), 한국어 자료는 5-way (+ KCI Open API). dual-write 정책: verification_log.md (불변 이력) + review_queue.md (action 큐). 위키 파일 자동 수정 없음.

→ .claude/agents/wiki-verifier.md
kci_helper.js (NEW)

KCI Open API for Korean papers한국어 자료용 KCI Open API

Korean Citation Index (KCI) integration for Korean-language journals (지방정부연구, 행정논총, 대한환경공학회지 etc.). crosscheck mode calls KCI + OpenAlex + Crossref simultaneously and auto-diffs discrepancies. Cache-share pattern: IP-whitelist constraint bypassed via cached XML committed to git — cloud environments read cache-only.한국 학술지 KCI Open API 통합 (지방정부연구·행정논총·대한환경공학회지 등). crosscheck 모드는 KCI + OpenAlex + Crossref를 동시 호출 + discrepancies 자동 diff. 캐시-공유 패턴: IP whitelist 제약을 git committed XML 캐시로 우회 — 클라우드 환경은 cache-only 모드.

Design principles설계 원칙

Four rules that keep Claude honest.Claude를 정직하게 유지하는 4가지 규칙.

These rules are enforced in CLAUDE.md and apply to every response, including synthesis pages.

이 규칙들은 CLAUDE.md에 명시되어 있으며, 개요 페이지 포함 모든 응답에 적용됩니다.

1

No web search웹 검색 금지

Claude never uses WebSearch or WebFetch to fill content gaps. Every answer comes from papers you actually have. If a paper is missing, Claude says so.Claude는 내용 공백을 채우기 위해 WebSearch나 WebFetch를 사용하지 않습니다. 모든 답변은 실제 보유한 논문에서 나옵니다. 논문이 없으면 없다고 말합니다.

2

Answer from the wiki first위키에서 먼저 답한다

sources/ and wiki/ are the only answer sources. Claude reads them before responding — not its training data.sources/와 wiki/가 유일한 답변 출처입니다. Claude는 학습 데이터가 아니라 이 파일들을 읽고 응답합니다.

3

If wiki is insufficient, re-read the PDF위키가 부족하면 PDF를 다시 읽는다

Claude goes back to papers/{stem}.pdf and extracts more detail with pypdf. Then updates the wiki with what it found.papers/{stem}.pdf로 돌아가 pypdf로 더 많은 내용을 추출합니다. 그런 다음 발견한 내용으로 위키를 업데이트합니다.

4

If no paper exists on the topic, say so해당 논문이 없으면 없다고 말한다

"I don't have a paper on this — please give me the PDF." No improvising. No confident wrong answers."이 주제에 대한 논문이 없습니다 — PDF를 주세요." 즉흥으로 만들어내지 않습니다. 자신감 있는 오답도 없습니다.
